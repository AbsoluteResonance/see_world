## 一、SLAM3R 部署

### 1.1 环境配置

**环境**: CPU-only 开发机 (AutoDL), CUDA Toolkit 11.8 已安装但无 NVIDIA 驱动。

| 问题 | 解决 |
|------|------|
| 开发环境无 GPU | 所有 GPU 依赖代码预留 TODO + stub 输出，待 GPU 环境切换后填充 |
| Python 3.12 基础环境 | 创建独立 `slam3r` conda 环境 (Python 3.11) 与 Phase 01 环境隔离 |

### 1.2 模型权重下载

| 问题 | 解决 |
|------|------|
| HuggingFace 权重需要 GPU 验证 | I2P/L2W 模型路径配置化: `config.py` 新增 `SLAM3R_MODEL_I2P` / `SLAM3R_MODEL_L2W` |

### 1.3 离线重建

| 问题 | 解决 |
|------|------|
| 无 GPU 无法运行 SLAM3R | `node.py` 实现 stub PLY 生成（彩色立方体），`slam3r_service.py` 标记 stub 状态，前端正常显示 |

### 1.4 在线重建

| 问题 | 解决 |
|------|------|
| 浏览器相机帧 → WebSocket → 后端推理全链路 | slam3r.js WebSocket 客户端 + slam3r_routes.py WebSocket 服务端已实现，帧率控制（每 3 帧发 1 帧） |

### 1.5 CUDA / 性能

| 问题 | 解决 |
|------|------|
| 无 GPU 时无法测试流式重建链路 | `slam3r_service.py` `process_frame()` 生成螺旋形 stub 点云（每帧 20-80 点增量增长），配合模拟相机位姿，实现无 GPU 前端渲染测试 |
| GPU 环境下 S LAM3R 模型加载 | `Image2PointsModel.from_pretrained('siyan824/slam3r_i2p')` 从 HuggingFace Hub 自动下载权重；共享单例避免多 stream 重复加载 |
| 离线重建流程 (视频 → 点云) | 直接调用 `scene_recon_pipeline_offline()` 执行完整 pipeline（关键帧采样 → I2P 局部重建 → L2W 全局配准） |
| 实时流推理 | 每 3 帧触发一次 I2P 推理，缓冲最近 10 帧作为上下文，输出置信度过滤后的 3D 点云 |

## 二、ROS2 节点 (slam3r_ros)

### 2.1 包创建与编译

| 问题 | 解决 |
|------|------|
| `setup.cfg` 包含 `install_dir` 导致 `ament_python` 构建失败 | 完全删除 `setup.cfg`，`ament_python` 不需要该文件，参照 `orb_slam3_ros` |
| 包路径在项目目录但 `colcon build` 需要在 ROS2 workspace | 软链接: `ln -sf /autodl-fs/.../slam3r_ros /root/ros2_ws/src/slam3r_ros` |

### 2.2 Topic 通信 (PointCloud2)

| 问题 | 解决 |
|------|------|
| PointCloud2 消息构造复杂 | 实现 `_make_pointcloud2()` 工具方法，支持 xyz + RGB 打包为标准 PCL 格式 |

### 2.3 SLAM3R 模型集成

| 问题 | 解决 |
|------|------|
| GPU 不可用时 SLAM3R import 失败 | 使用 `_try_import_slam3r()` 惰性导入 + 异常捕获，无 GPU 时生成 stub 数据 |

|------|------|

### 4.1 稠密点云渲染 (增量更新)

| 问题 | 解决 |
|------|------|
| 实时流场景需要逐批添加点而非全量加载 | `viewer_dense.js` 新增 `addPoints()` 方法，每次调用创建新的 Points 对象加入 Group |

### 4.2 WebSocket 前端对接

| 问题 | 解决 |
|------|------|
| `app.js` 的 `currentAnalysisFileId` 是闭包变量，`slam3r.js` 无法访问 | 在 `openAnalysis()` 中设置 `window._currentAnalysisFileId = fileId` |

| 问题 | 解决 |

## 七、技术决策记录

| 决策 | 理由 | 日期 |
|------|------|------|
| 无 GPU 时生成 stub PLY 点云 | 确保前端到后端全链路可测试，不阻塞开发进度 | 2026-05-23 |
| SLAM3R 前端独立为 `slam3r.js` | 与 Phase 01 `app.js` 解耦，不修改现有 ORB-SLAM3 交互逻辑 | 2026-05-23 |
| 帧率控制 3:1 发送 | 浏览器 60fps 过高，GPU 推理通常 5-10fps，3 帧选 1 帧合理 | 2026-05-23 |
| ROS2 节点三模式 (batch/video/serve) | batch 供 Web 后端调用，video 供直接测试，serve 供实时流 | 2026-05-23 |
| `slam3r_ros` 惰性导入 SLAM3R | 使用惰性导入 + 异常捕获，ROS2 节点即使在无 GPU 环境也能启动 | 2026-05-23 |
| 共享模型单例 (`_shared_i2p_model`/`_shared_l2w_model`) | I2P/L2W 模型 ~1GB 显存，多个 stream 共享同一实例避免 OOM | 2026-05-23 |
| 离线推理优先走 Python API 而非 ROS2 子进程 | 直接调用 `scene_recon_pipeline_offline` 避免 subprocess 开销和序列化延迟 | 2026-05-23 |
