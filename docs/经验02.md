# See World — 开发经验总结 (Phase 02)

> 记录 Phase 02 开发过程中遇到的关键问题、解决方案和技术决策。
> 创建日期: 2026-05-23 | 最后更新: 2026-05-24

---

## 一、架构演进

### 原始计划: SLAM3R + ROS2 + Web
- SLAM3R (I2P/L2W) 稠密重建 → ROS2 节点 → Web 前端
- 依赖 HuggingFace 模型下载，需要 GPU

### 实际方案: MASt3R-SLAM + HTTP API + Web/Android
- MASt3R-SLAM (实时稠密 SLAM) → HTTP POST 帧 → 点云轮询
- 无 ROS2，纯 Python 子进程通信
- 前端: HTTP fetch 替代 WebSocket（cloudflare 隧道对 WebSocket 不稳定）

### 移除的组件
- SLAM3R (模型太大，HuggingFace 下载不稳定)
- ROS2 (Python 3.10 限制，增加复杂度无实际收益)
- VINS-Mono (IMU+VO，效果差)

## 二、关键技术问题

### 2.1 WebSocket vs HTTP POST

| 问题 | 方案 |
|------|------|
| cloudflared 隧道 WebSocket 连接不稳定，经常断开 | 改用 HTTP POST 每帧独立请求，fire-and-forget |
| 点云响应太大 (~2MB)，隧道传不回来 | 拆分 `/frame`（轻量确认）和 `/points`（单独拉取）两张端点 |
| gzip 压缩 | 5000 点 JSON ~170KB → 压缩后 ~30KB |

### 2.2 3D 查看器问题

| 问题 | 解决方案 |
|------|----------|
| `denseViewer.init()` 抛异常导致帧循环不启动 | 用 try-catch 包裹 init() 调用 |
| `statusEl` 闭包引用失败 | 改为函数内部 `document.getElementById()` 重新获取 |
| `canvas.toBlob` + `FileReader` 在 X 浏览器上不稳定 | 改用 `canvas.toDataURL()` 同步转换 |
| Service Worker 缓存旧 JS | 添加 `Cache-Control: no-store` 头 + 强制 unregister |
| Y 轴/ Z 轴方向错误 | 在推理子进程中取反 Y 和 Z 坐标 |

### 2.3 MASt3R-SLAM 推理问题

| 问题 | 解决方案 |
|------|----------|
| tracker 不产生新关键帧，点云不更新 | 强制每 3 帧添加新关键帧 |
| `Boolean value of Tensor with more than one value is ambiguous` | `match_frac = valid_opt.sum().item() / valid_opt.numel()` |
| 点云只从最新关键帧提取 | 改为从所有关键帧提取，每帧取 `pts_per_kf` 个点 |

### 2.4 帧传输与保存

| 问题 | 解决方案 |
|------|----------|
| 帧传输不稳定，不知帧是否到达服务器 | 创建独立帧测试页 `/static/frame_test.html` |
| 需要查看实际收到的帧 | 添加"保存帧"复选框，帧存到 `/tmp/mast3r_frame_*.jpg` |

### 2.5 服务端问题

| 问题 | 解决方案 |
|------|----------|
| `time.sleep(0.2)` 阻塞事件循环，后续请求排不上 | 改为 `await asyncio.sleep(0.2)` |
| 模型加载慢（~30s） | 启动时预加载推理子进程 |

## 三、Android APP 构建

### 编译流程
- 项目: 最小 WebView APP，加载 MASt3R-SLAM 前端
- CI: GitHub Actions + `android-actions/setup-android@v3`
- Gradle 8.5 + compileSdk 34 + AndroidX

### 踩过的坑

| 问题 | 解决 |
|------|------|
| `gradle-wrapper.jar` 被 .gitignore 排除 | 从 .gitignore 移除并 `git add -f` 强制提交 |
| `gradlew` 的 `$APP_HOME` 未设置 | 添加路径解析逻辑 |
| Kotlin DSL: `include ':app'` 错误 | 改为 `include(":app")` |
| AndroidX 依赖未启用 | 创建 `gradle.properties` 添加 `android.useAndroidX=true` |
| Release 创建 403 | 添加 `permissions: contents: write` |
| APK 文件名不匹配 | 用 `cp` 重命名为 `SeeWorld-v1.0.0.apk` |

## 四、最终架构

```
手机相机 → canvas.toDataURL → HTTP POST → Python 3.11
  → MASt3R-SLAM 推理 → 点云 JSON → 返回前端 → Three.js 渲染

启动: python start_mast3r.py  (自动加载模型 + 启动服务器)
访问: cloudflared tunnel → trycloudflare.com 域名
Android: GitHub Releases 下载 APK
```

### 性能
- 每帧推理: ~500ms (GPU)
- 每帧点数: 2000
- 帧间隔: 1.5s (前端) / 3s (点云轮询)
- 隧道延迟: ~5s (cloudflare 往返)
