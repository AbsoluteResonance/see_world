## 一、SLAM 编译与部署

### 1.1 ORB-SLAM3 编译问题

**Python 版本**：Ubuntu 22.04, Python 3.12, OpenCV 4.5.4, Eigen3 3.4.0, Boost 1.74。

**问题：`Examples_old/` 目录缺失**
ORB-SLAM3 `--depth=1` clone 时没有 `Examples_old/`，但 CMakeLists.txt 引用了它。
- **解决**：删除 CMakeLists.txt 中 `Examples_old/` 相关行

**问题：`std::chrono::monotonic_clock` 不存在**
C++11 早期草案名称，后被重命名为 `steady_clock`。
- **解决**：全局替换为 `std::chrono::steady_clock`

**问题：Pangolin 需要 C++14**
Pangolin 依赖 sigslot 库使用 C++14 特性，ORB-SLAM3 设了 `-std=c++11`。
- **解决**：升级到 `-std=c++14`

**问题：Pangolin 需要 X Display / 关闭时 SIGABRT**
无 GUI 服务器上 Pangolin 可视化 crash，析构时也有内存问题。
- **解决**：禁用可视化 `System(..., MONOCULAR, false)`, 安装 xvfb
- 轨迹保存提前在 shutdown 前执行，或在每帧 `TrackMonocular` 后直接保存

**问题：轨迹保存空文件**
`SaveKeyFrameTrajectoryTUM` 只从当前活跃地图读取关键帧。地图重置(`clearMap`)后关键帧丢失。
- **解决**：在 `ResetActiveMap` 中 `clearMap` 前调用 `SaveKeyFrameTrajectoryTUM`
- 同时修改 `SaveKeyFrameTrajectoryTUM` 遍历所有地图而非仅当前地图

**问题：单目 SLAM 追踪失败**
对于手机拍摄的室内视频，ORB-SLAM3 单目模式需要显著的相机平移运动才能初始化。
帧与帧之间特征匹配不足导致 "Fail to track local map" 和反复重置。
- **结论**：ORB-SLAM3 对这类数据不适用，需换用更适合的 SLAM 方案

### 1.2 VINS-Mono

**VINS-Mono 需要 ROS1 (Noetic)，不是 ROS2。**
- Ubuntu 22.04 上可安装 ROS2 Humble，但 VINS-Mono 使用 ROS1 API (roscpp, tf, cv_bridge)
- ROS1 Noetic 与 Ubuntu 22.04 的库 (libboost1.71 等) 不兼容
- 如需 VINS-Mono：在 Docker 中运行 `osrf/ros:noetic-desktop-full`

### 1.3 相机标定

- 标定功能已集成到 Web 端：`/calibrate.html`
- 上传棋盘格照片 → OpenCV 检测角点 → 计算内参 → 生成 ORB-SLAM3 格式 YAML
- 支持缩放到 SLAM 使用的尺寸：`/api/calibrate/result/yaml?target_width=640&target_height=480`
- 棋盘格在线页面：`/checkerboard.html`（全屏显示，手机拍照即可）
- 默认参数：9×6 内角点，格子大小可自定义

## 三、Web 前端

### 3.1 上传按钮双击文件选择器

**问题**：`<label for="fileInput">` 和 `dropzone.click` 同时触发文件选择器。
- **解决**：检测点击目标，label 按钮跳过 `fileInput.click()`

### 3.2 上传进度 100% 卡住

**问题**：画廊显示了 `.gitkeep` 隐藏文件导致渲染错误。
- **解决**：后端 `/api/files` 过滤 `.` 开头的文件

### 3.3 Service Worker 缓存

**问题**：PWA 缓存旧 JS 文件，修改后不更新。
- **解决**：`CACHE_NAME` 版本号 + JS 引用 `?v=N`

### 3.4 文件重命名/删除

- `PUT /api/files/{id}/rename` — 重命名文件
- `DELETE /api/files/{id}` — 删除文件
- 前端分析面板底部操作按钮

### 3.5 视频缩略图

- 后端用 OpenCV 提取视频首帧，缓存为 JPEG 缩略图
- 前端 gallery 用 `poster="/api/files/{id}/thumbnail"` 显示

## 四、外网隧道

### 4.1 Serveo (弃用)

- 纯 SSH 隧道，无需安装
- 免费版有连接超时限制（~10s），不适合 Kimi API 和大文件上传

### 4.2 Cloudflare Tunnel (推荐)

- 无带宽/超时限制，自动 HTTPS
- `cloudflared tunnel --url http://localhost:PORT`
- 免费版域名每次重启会变

## 五、后端架构

### 5.1 导入路径

- `main.py` 启动时把 `see_world/` 根目录加入 `sys.path`
- `slam_service.py` 动态导入 `SLAM.ORB_SLAM3.python.SLAMRunner`
- `__file__` 路径决定 workspace，进而决定默认参数路径

### 5.2 Job 管理

- 内存 dict 存储任务状态（重启丢失）
- 状态机：`pending → preparing/extracting → running → completed/failed`
- 前端 1s 轮询进度
- 新增 `POST /api/reconstruct/from-file/{file_id}` 端点上传文件直接重建

### 5.3 错误码

| 错误码 | 含义 |
|--------|------|
| 0 | 成功 |
| 40001 | 参数错误 |
| 40002 | 文件格式不支持 |
| 40003 | 文件过大 |
| 404 | 资源不存在 |
| 50001 | 服务内部错误 |
| 50002 | Kimi API 调用失败 |
| 50003 | SLAM 模型错误 |
